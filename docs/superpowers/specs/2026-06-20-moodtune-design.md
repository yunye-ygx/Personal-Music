# MoodTune — 私人音乐助手 设计文档

## 1. 产品定义

**目标用户**：自己（单用户 MVP）

**核心价值**：通过对话理解用户此刻的心情，从个人歌单中推荐一首恰到好处的歌。用户不一定知道自己想听什么，但听到对的歌会静下心来。

**一句话**：一个能理解你心情的私人音乐顾问。

## 2. MVP 范围

### 做

- 聊天界面（文字输入，流式输出）
- AI 从对话中理解心情，从个人歌单推荐歌曲
- 展示推荐卡片：歌名 + 歌手 + 推荐理由
- 点击后在应用内播放（MinIO URL）
- 通过歌单链接导入歌曲
- 歌曲分组管理（导入分组 + 手动分组）

### 不做（后续迭代）

- 语音识别
- 天气等外部工具调用
- 歌词、封面展示
- 拓展推荐新歌
- 推荐反馈追踪（喜欢/跳过）
- RAG 向量检索（但数据库预留 pgvector）
- 多用户 / 登录

## 3. 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 |
| 后端 | Java / Spring Boot |
| 数据库 | PostgreSQL + pgvector 扩展 |
| 文件存储 | MinIO（歌曲 MP3） |
| 缓存 | Redis |
| AI | 国内大模型 API（通义千问 / DeepSeek 等） |
| 实时通信 | SSE（Server-Sent Events）流式输出 |

## 4. 核心交互流程

```
用户打开页面
    ↓
左侧显示会话列表，可新建会话
    ↓
用户在聊天框输入文字（如："今天好累，加班到现在"）
    ↓
后端通过 SSE 流式调用大模型
    ↓
AI 返回结构化数据：文字回复 + 推荐歌曲信息
    ↓
前端渲染：气泡文字 + 歌曲卡片（歌名、歌手、推荐理由）
    ↓
用户点击卡片 → 跳转播放页面，通过 MinIO URL 播放
    ↓
用户可继续聊天："换一首更安静的"
```

## 5. LLM 调用设计

### 5.1 Prompt 角色设定

AI 角色：用户的知心朋友、私人音乐顾问。根据对话内容和上下文推荐歌曲。

### 5.2 输入

- 系统提示词（角色设定 + 可用歌曲列表）
- 本次会话的聊天历史
- 当前时间

### 5.3 输出（结构化）

```json
{
  "text": "安静的夜晚很适合这首歌",
  "song_id": 123
}
```

- `text`：AI 的对话内容（推荐理由自然融入其中，不单独设置心情/天气字段）
- `song_id`：推荐的歌曲 ID（关联 songs 表，null 表示纯对话无推荐）

### 5.4 推荐逻辑

- 将用户歌单信息作为上下文传入 LLM
- LLM 根据对话内容判断心情，从歌单中选择最匹配的歌曲
- 推荐理由自然地写在 text 中（如涉及心情/天气，自然融入而非强制）

### 5.5 "换一首" 机制

用户说"换一首"时，聊天历史中已推荐的歌曲自动作为上下文，AI 避免重复推荐并调整方向。

### 模块职责

| 模块 | 职责 |
|------|------|
| 聊天服务 | SSE 流式通信，管理会话和消息，调用推荐引擎 |
| 歌库服务 | 歌曲 CRUD、歌单链接解析导入、分组管理 |
| 推荐引擎 | 组装 Prompt（角色设定 + 歌单列表 + 聊天历史），调用 LLM，解析结构化返回 |

## 7. 数据模型

### 7.1 songs — 歌曲表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| title | VARCHAR | 歌曲名称 |
| artist | VARCHAR | 歌手 |
| genre | VARCHAR | 风格标签（民谣、摇滚、轻音乐等） |
| file_url | VARCHAR | MinIO 文件地址 |
| liked | BOOLEAN | 是否标记为喜欢 |
| created_at | TIMESTAMP | 创建时间 |

### 7.2 song_groups — 歌曲分组表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| name | VARCHAR | 分组名称 |
| source | VARCHAR | 来源：imported（导入）/ manual（手动） |
| created_at | TIMESTAMP | 创建时间 |

### 7.3 song_group_relations — 歌曲-分组关联表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| song_id | BIGINT FK | 关联歌曲 |
| group_id | BIGINT FK | 关联分组 |

一首歌可属于多个分组（多对多）。

### 7.4 song_imports — 导入记录表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| source_link | VARCHAR | 歌单链接 |
| platform | VARCHAR | 来源平台（qq_music / netease 等） |
| status | VARCHAR | 状态：pending / success / failed |
| imported_count | INTEGER | 已导入歌曲数 |
| created_at | TIMESTAMP | 创建时间 |

### 7.5 chat_sessions — 聊天会话表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| title | VARCHAR | 会话标题（首条消息） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 最后活跃时间 |

### 7.6 chat_messages — 聊天消息表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键 |
| session_id | BIGINT FK | 所属会话 |
| sender_type | VARCHAR | 发送者：user / ai |
| text_content | TEXT | 消息文字内容 |
| song_id | BIGINT FK (nullable) | AI 推荐的歌曲，用户消息为 null |
| created_at | TIMESTAMP | 创建时间 |

## 8. API 设计（关键接口）

### 聊天

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/sessions | 新建会话 |
| GET | /api/sessions | 获取会话列表 |
| GET | /api/sessions/{id}/messages | 获取会话聊天记录 |
| POST | /api/sessions/{id}/messages | 发送消息，返回 SSE 流 |

### 歌库

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/songs | 获取歌曲列表（支持分组筛选） |
| POST | /api/songs | 手动添加歌曲（含上传文件到 MinIO） |
| POST | /api/songs/import | 通过歌单链接导入 |
| GET | /api/song-groups | 获取分组列表 |
| POST | /api/song-groups | 新建分组 |

### 播放

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/songs/{id}/stream | 获取 MinIO 播放地址（或代理流） |

## 9. 前端页面

| 页面 | 说明 |
|------|------|
| 主页（聊天） | 左侧会话列表 + 右侧聊天区域，AI 回复含歌曲卡片 |

| 播放页 | 点击歌曲卡片后进入，播放器 + 歌曲信息 |
